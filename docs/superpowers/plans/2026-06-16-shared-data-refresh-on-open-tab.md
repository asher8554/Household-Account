# 2026-06-16 공유 데이터 열린 탭 재동기화 계획

## 문제

아내 휴대폰에서 거래 카테고리를 `장보기`에서 `육아`로 바꿔 GitHub 공유 파일 push가 성공했지만, PC 화면의 달력에는 이전 카테고리가 계속 보인다.

## 확인한 사실

- GitHub Pages 공유 파일 `shared-data.json`은 `2026-06-16T02:48:14.343Z`로 갱신됐다.
- 최신 공유 파일에는 `장보기 -> 육아`로 바뀐 거래가 포함되어 있다.
- 앱은 시작 시 한 번만 `loadPublishedSharedData()`를 호출한다.
- 이미 열린 PC 탭은 휴대폰 push 이후 원격 공유 파일을 다시 읽지 않는다.

## 원인 가설

PC 탭이 휴대폰 push 전에 이미 열려 있으면, 앱이 새 `shared-data.json`을 다시 가져오지 않아 IndexedDB와 달력 집계가 이전 상태로 남는다.

## 구현 방향

1. 공유 데이터 동기화를 재사용 가능한 hook으로 분리한다.
2. 앱 시작 시 기존처럼 한 번 동기화한다.
3. 창이 다시 focus 되거나 탭이 visible 상태가 될 때 공유 데이터를 다시 확인한다.
4. 너무 잦은 GitHub Pages fetch를 막기 위해 짧은 최소 간격을 둔다.
5. 테스트로 focus/visibility 이벤트가 재동기화를 호출하는지 고정한다.

## 검증

- 관련 단위 테스트 또는 소스 보호 테스트를 추가한다.
- 전체 Playwright 테스트를 실행한다.
- `npm run build`를 실행한다.
- main에 push한 뒤 GitHub Pages 배포 성공과 live 번들 반영을 확인한다.
